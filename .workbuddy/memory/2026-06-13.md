## PDF 导出修复（`$$` display math 追踪）

### 问题
用户导出 PDF 时报 `Missing $ inserted`、`Undefined control sequence \sqr`、`\begin{cases} ended by \end{solution}` 等错误。

根因：`sanitizeAnswer()` 和 `removeRawMathOutsideEnv()` 没有追踪 `$$...$$` display math 环境，导致：
1. `$$` 被误判为两个独立的 `$`（inline math）
2. `$$...$$` 块内的 `cases` 等环境状态混乱
3. 未闭合的 `$$` 导致后续内容被错误地处理

### 修复（server.cjs）

#### 1. `sanitizeAnswer()` 栈扫描器
- 在 `$` 检测之前添加 `$$` 检测（`$$` 必须优先匹配）
- 新增 `displaydollar` 类型入栈/出栈
- 栈中类型列表：`inline`、`display`、`displaydollar`、`dollar`、`leftp`、`leftb`

#### 2. `removeRawMathOutsideEnv()` 状态追踪
- 新增 `inDollarDisplay` 追踪 `$$...$$`
- 新增 `inDollarInline` 追踪 `$...$`
- 计数规则：`$$` 个数奇数→切换状态；去掉 `$$` 后统计剩余 `$` 个数奇数→切换状态
- 四种数学环境内均保留整行（不执行裸露命令检查）

## 选择题排版修复（根因：正则不匹配导致选项提取失败）

### 问题（4项）
1. 表头缺少 姓名/得分/考试时间
2. 选择题选项和题目混在同一行
3. 选项间无间隔
4. `（）` 括号内无空间

### 根因分析
**`getQuestionFormat()` (line 713)** 用旧正则 `/ [A-D]\.\s/` 判断选择题：
- 要求字母前有空格 ` ` + 字母后有空格 `\s`
- 实际数据格式：`\n\nA．中文文字` （换行+全角句号+无尾部空格）
- 正则不匹配 → 返回 `'solution'` 而非 `'single'` → **完全跳过选项格式化代码**
- 同样的问题存在于 Way B 提取正则

### 修复（3处同步更新）
1. **`getQuestionFormat()`**: `/(?:^|\s)([A-D])[\.．]/`
2. **Way B 提取**: 同上正则，分割用 `split(/\s(?=[A-D][\.．])/)`
3. **表头**: 改为 `\noindent 姓名：\rule{2.5cm}{0.4pt}\hfill 得分：\rule{2cm}{0.4pt}`（左对齐姓名/右对齐得分）
4. **括号间距**: `\quad` → `\qquad`（双倍宽度）
5. **选项换行**: 题干后 `\par\vspace{0.3em}` 再接 choices 环境

### 教训
- 选项检测的正则出现在 **3 个地方**，修改时必须全部同步
- 全角句号 `．`(U+FF0E) 和换行符 `\n` 是常见的数据格式差异
- 当 LaTeX 输出与预期不符时，先检查函数返回值（debug log），而非猜测
